01
데이터로 일한다는 건 뭘까?

우리가 매일 하는 일들, 고객 문의를 처리하거나, 계약서를 작성하거나, 공간을 운영하는 것들. 이 모든 활동은 흔적을 남깁니다. 언제 계약했는지, 어떤 공간이 자주 쓰이는지, 어떤 문의가 많은지. 이게 바로 데이터입니다.

예전엔 이 흔적들을 모아 정리하는 게 번거로워서 감이나 경험에 의존해 결정을 내리는 경우가 많았어요. 하지만 지금은 이 흔적들이 모두 자동으로 기록되고 저장됩니다. 데이터로 일한다는 건, 이 기록들을 직접 꺼내서 확인하고 판단에 활용한다는 뜻이에요.

핵심 포인트

"왜 이번 달 계약이 줄었을까?" 이런 질문에 느낌이 아닌 숫자로 답할 수 있게 되는 것. 그게 데이터로 일하는 출발점입니다.

02
SQL이 뭘까?

SQL은 데이터베이스에서 원하는 정보를 꺼내기 위한 언어입니다. 프로그래밍 언어처럼 보이지만, 사실 영어 문장에 매우 가깝게 생겼어요.

인스타그램이나 카카오톡을 생각해볼게요. 내 친구 목록, 내가 올린 사진들, 주고받은 메시지들이 어딘가에 저장되어 있어야 앱을 켤 때마다 볼 수 있겠죠? 이처럼 수많은 정보가 엑셀 표처럼 정리된 거대한 디지털 창고를 데이터베이스(Database)라고 부릅니다.

그런데 이 데이터 관리자(컴퓨터)에게 한국어로 "서울 사는 고객들만 찾아줘!"라고 말하면 알아듣지 못해요. 컴퓨터가 정확히 이해할 수 있도록 규칙을 정해 만든 주문서 작성법이 바로 SQL입니다.

비유로 이해하기
도서관에서 사서에게 "2024년에 출판된 경제 책 중에서 대출이 많은 순서로 5권만 보여주세요"라고 요청하는 것. SQL은 바로 그 요청문입니다. 사서(데이터베이스)는 요청을 이해하고 결과를 가져다줍니다.
실제 SQL이 이렇게 생겼어요
SELECT 이름, 지역, 계약일
FROM 고객명부
WHERE 지역 = '서울'
ORDER BY 계약일 DESC;
→ "고객명부에서, 서울에 사는 고객의 이름·지역·계약일을 가져오되, 최근 계약 순서로 보여줘"

어때요? 영어를 조금만 알면 무슨 뜻인지 대충 유추할 수 있을 정도로 꽤 직관적이죠? 코딩처럼 외계어 같은 게 아니라, 영어 문장과 아주 비슷한 구조예요.

엑셀로 하면
파일 열고 → 필터 설정하고 → 정렬하고 → 복사·붙여넣기. 데이터가 크면 느려지고, 매번 반복해야 합니다.
SQL로 하면
한 번 작성해두면 언제든 다시 실행. 수백만 건도 순식간. 조건만 바꾸면 다른 분석도 바로 가능합니다.
03
SQL로 할 수 있는 4가지 핵심 작업

SQL을 이용하면 데이터 창고에서 딱 4가지 주요 작업을 할 수 있어요. 우리 업무에서 가장 자주 쓸 것은 첫 번째, 검색(SELECT)입니다.

SELECT
🔍 검색하기 — 제일 많이 씀!
창고에서 원하는 정보만 쏙쏙 골라서 찾아옵니다.
예) "이번 달 계약한 고객 목록 보여줘"
INSERT
➕ 추가하기
새로운 데이터를 창고에 집어넣습니다.
예) 신규 회원가입 시 정보 등록

UPDATE
✏️ 수정하기
이미 있는 데이터를 새로운 내용으로 고칩니다.
예) 고객 주소 변경 처리
DELETE
🗑️ 삭제하기
필요 없는 데이터를 완전히 없앱니다.
예) 탈퇴 회원 정보 삭제
실무 포인트

우리가 주로 배울 건 SELECT, 즉 조회(검색)입니다. INSERT·UPDATE·DELETE는 데이터를 건드리기 때문에 함부로 실행하면 위험해서, 보통 개발팀이 담당합니다. 우리 목표는 데이터를 읽고 분석하는 것이에요.

04
빅쿼리(BigQuery)는 뭘까?

빅쿼리는 구글이 만든 데이터 분석 도구입니다. 쉽게 말하면, SQL을 입력하는 창이 있고, 거기에 질문을 쓰면 엄청난 양의 데이터에서 답을 뽑아주는 서비스예요.

우리가 구글 드라이브를 쓰는 것처럼, 빅쿼리도 웹 브라우저에서 바로 씁니다. 설치도 없고, 내 컴퓨터 성능과도 무관합니다. 데이터가 수백만 줄이어도 몇 초 안에 결과가 나옵니다.

비유로 이해하기
엑셀이 내 책상 위의 계산기라면, 빅쿼리는 초고속 슈퍼컴퓨터에 연결된 검색창입니다. 규모가 커질수록 진가가 드러납니다.
웹 브라우저에서 실행 설치 불필요 수백만 건도 수초 내 처리 SQL 문법 그대로 사용 구글 계정으로 접속
🚀
상상 이상의 속도
수백억 줄의 데이터에서 통계를 내는 데 고작 몇 초밖에 안 걸려요. 구글의 컴퓨터들이 일을 잘게 쪼개서 동시에 처리하기 때문입니다.
🗣️
새로운 언어를 배울 필요 없음
앞에서 배운 SQL 문법을 그대로 입력하면 됩니다. 빅쿼리 전용 언어 같은 건 없어요.
☁️
내 컴퓨터 성능과 무관
구글 클라우드에서 처리되기 때문에 내 노트북이 느려도 전혀 상관없습니다. 웹 브라우저만 있으면 됩니다.
한 줄 정리
SQL은 데이터를 다루는 말(언어)이고, 빅쿼리는 그 SQL을 찰떡같이 알아듣고 엄청 큰 데이터도 순식간에 찾아주는 구글의 초고속 만능 창고입니다.
05
왜 배워야 할까? 실무에서 어떻게 쓸까?

SQL을 쓸 수 있으면, 누군가에게 부탁하지 않아도 궁금한 것을 바로 확인할 수 있습니다. 데이터팀이나 개발팀에 요청하고 기다리는 시간이 줄어들고, 내가 원하는 방식으로 원하는 시점에 숫자를 볼 수 있어요.

영업 / 운영
이번 달 지점별 계약 현황 직접 뽑기. 특정 고객군의 계약 패턴 파악.
마케팅 / 기획
캠페인 전후 수치 비교. 유입 채널별 전환율 분석.

HR / 총무
부서별 교육 이수 현황. 공간 이용률 집계.
PM / 서비스 기획
기능별 사용 빈도 확인. 특정 이벤트 발생 추이 파악.
이런 순간에 씁니다

"지난 3개월 동안 계약 해지한 고객 중에, 입주 기간이 6개월 미만이었던 비율이 어떻게 돼?"
이런 질문, 지금은 누군가에게 부탁해야 하지만 SQL을 알면 내가 직접 5분 안에 답할 수 있습니다.

06
쉽게 시작하는 공부법

SQL은 외우는 공부가 아닙니다. 자주 쓰는 명령어는 10개 내외이고, 나머지는 필요할 때 찾아쓰면 돼요. 공부의 핵심은 실제 데이터로 직접 써보는 것입니다.

1
개념보다 핵심 문법 4개 먼저 눈에 익히기
SELECT, FROM, WHERE, ORDER BY. 이 네 가지만 알아도 웬만한 조회는 다 됩니다. 교육 전에 이 단어들을 한 번 검색해보세요.
2
직접 써보기 — 읽는 것의 10배 효과
빅쿼리 샌드박스(무료)를 열고, 예제 쿼리를 그대로 입력해보세요. 틀려도 됩니다. 오류 메시지가 가장 좋은 선생님입니다.
3
내 질문을 SQL로 번역해보기
평소에 궁금했던 것 하나를 정하고, 그걸 SQL로 표현하려고 해보세요. 실제 업무와 연결될수록 기억에 훨씬 오래 남습니다.
4
모르면 AI에게 바로 물어보기
Gemini나 ChatGPT에게 "이런 결과를 보려면 SQL을 어떻게 써야 해?"라고 물어보면 바로 답을 줍니다. 구글 검색보다 훨씬 빠르고 구체적이에요.
07
교육 전에 알아두면 좋은 용어들

뜻만 알아도 흐름을 따라가기 훨씬 쉬워요.

Table
테이블
데이터가 담긴 표. 엑셀 시트랑 비슷한 개념이에요.
Row
행 (가로 줄)
테이블의 가로 한 줄. 한 건의 데이터. 예) 계약 1건.
Column
열 (세로 항목)
테이블의 세로 항목. 예) 고객명, 계약일, 금액.
Query
쿼리
데이터베이스에 보내는 질문. SQL로 작성합니다.
Database
데이터베이스
테이블들이 저장된 창고. 빅쿼리가 이 창고 역할을 해요.
Aggregation
집계
여러 값을 요약하는 것. 합계·평균·개수 등이 여기에 해당해요.
