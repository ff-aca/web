FF Academy Week 1 - 보충자료

생소한 SQL,
이 순서대로만 따라오면 됩니다

SQL은 결국 정해진 순서대로 키워드를 나열하는 것뿐입니다.
아래 8단계가 BigQuery에서 SQL 문장을 "쓰는 순서"입니다.

SELECT› FROM› JOIN› WHERE› GROUP BY› HAVING› ORDER BY› LIMIT

필요한 키워드만 골라 쓰면 되고, 쓸 때는 반드시 이 순서를 지켜야 합니다.

개념

테이블 · 행 · 열이 뭔가요?

SQL은 표(테이블) 형태로 저장된 데이터를 다룹니다. 아래 students 테이블을 이 가이드 전체에서 예제로 사용합니다.

idnameclassscorecity
1민준A85서울
2서연B92부산
3도윤A78서울
4하은B95인천
5지호A60부산
6수아C88서울

가로 한 줄(예: 민준의 정보 전체)이 행(row), 세로 한 줄(예: score 전체)이 열(column)입니다.
BigQuery에서는 테이블을 프로젝트.데이터셋.테이블 형태의 전체 경로로 부르는 경우가 많은데, 아래 예제에서는 myproject.mydataset.students로 표기하겠습니다.

STEP 01 · 조회

SELECT / FROM — 데이터 조회하기

SELECT는 "어떤 열을 보고 싶은지", FROM은 "어느 테이블에서 가져올지"를 정합니다. BigQuery에서는 테이블명을 백틱(`)으로 감싼 프로젝트.데이터셋.테이블 경로로 씁니다.

googlesql 
SELECT name, score
FROM `myproject.mydataset.students`;
namescore
민준85
서연92
도윤78
하은95
지호60
수아88

모든 열을 다 보고 싶다면 "전부"라는 뜻의 *(별표)를 씁니다.

googlesql
SELECT *
FROM `myproject.mydataset.students`;
💰

BigQuery 실전 팁 — SELECT *는 모든 컬럼을 다 읽기 때문에 과금되는 스캔 데이터 양이 늘어납니다.
실무에서는 꼭 필요한 컬럼만 선택하는 습관을 들이는 게 좋아요.

STEP 02 · 연결

(LEFT) JOIN — 다른 테이블과 연결하기

JOIN은 공통된 값(키)을 기준으로 두 테이블을 옆으로 이어 붙여서 함께 조회하는 키워드입니다. 동아리 가입 정보가 담긴 clubs 테이블을 하나 더 추가해보겠습니다.

idstudent_idclub_name
11축구부
22미술부
34축구부

※ 도윤(3), 지호(5), 수아(6)는 동아리에 가입하지 않아 clubs 테이블에 없습니다.

LEFT JOIN은 왼쪽(기준) 테이블의 모든 행을 그대로 유지하면서, 오른쪽 테이블에 매칭되는 값이 없으면 그 자리를 NULL로 채웁니다.

googlesql
SELECT
  s.name,
  c.club_name
FROM `myproject.mydataset.students` AS s
LEFT JOIN `myproject.mydataset.clubs` AS c
  ON s.id = c.student_id;
nameclub_name
민준축구부
서연미술부
도윤NULL
하은축구부
지호NULL
수아NULL

AS s, AS c는 테이블에 짧은 별칭(alias)을 붙인 것입니다. 테이블명이 길어질 때 s.name처럼 "별칭.열이름"으로 간결하게 쓸 수 있습니다.

⚠️

같은 자리에 INNER JOIN을 쓰면 양쪽 모두에 매칭되는 값이 있는 행만 남아서, 동아리가 없는 도윤·지호·수아는 결과에서 아예 빠지게 됩니다. "전체를 다 보여주되 없으면 NULL"이 필요할 때 LEFT JOIN을 씁니다.

STEP 03 · 조건

WHERE — 조건으로 걸러내기

조건을 만족하는 행만 골라낼 때 사용합니다.

googlesql
SELECT name, score
FROM `myproject.mydataset.students`
WHERE score >= 85;
namescore
민준85
서연92
하은95
수아88

자주 쓰는 조건 연산자

연산자의미예시
=같다WHERE city = '서울'
!= / <>다르다WHERE city != '서울'
> < >= <=크다 / 작다WHERE score > 80
AND그리고 (둘 다 만족)WHERE class='A' AND score>70
OR또는 (하나만 만족)WHERE city='서울' OR city='부산'
BETWEEN범위 안에 있음WHERE score BETWEEN 80 AND 95
IN목록 중 하나WHERE city IN ('서울','인천')
IS NULL값이 비어있음WHERE city IS NULL

🔍 LIKE와 와일드카드 — 문자 패턴으로 찾기

LIKE는 정확히 일치하지 않아도, 비슷한 패턴의 문자열을 찾을 때 씁니다.
이때 와일드카드라는 특수 기호를 함께 사용합니다.

기호의미예시
%0개 이상의 임의의 문자LIKE '민%' → "민"으로 시작
_정확히 1개의 임의의 문자LIKE '_연' → 두 번째 글자가 "연"인 2글자
googlesql
-- "민"으로 시작하는 이름
SELECT name FROM `myproject.mydataset.students` WHERE name LIKE '민%';
-- 결과: 민준

-- "은"으로 끝나는 이름
SELECT name FROM `myproject.mydataset.students` WHERE name LIKE '%은';
-- 결과: 하은

-- "연"이 어디든 포함된 이름
SELECT name FROM `myproject.mydataset.students` WHERE name LIKE '%연%';
-- 결과: 서연

-- "은"으로 끝나지 않는 이름 (NOT LIKE)
SELECT name FROM `myproject.mydataset.students` WHERE name NOT LIKE '%은';
💡

%와 _는 함께 조합할 수도 있습니다. 예를 들어 LIKE '_%연%'처럼 자유롭게 섞어서 더 세밀한 패턴을 만들 수 있어요.

STEP 04 · 집계

GROUP BY — 그룹으로 묶어서 집계하기

같은 값을 가진 행끼리 묶어서, 집계 함수로 합계나 평균을 계산할 때 사용합니다.

함수의미
COUNT()개수
SUM()합계
AVG()평균
MAX() / MIN()최댓값 / 최솟값
googlesql
SELECT class, AVG(score) AS avg_score, COUNT(*) AS student_count
FROM `myproject.mydataset.students`
GROUP BY class;
classavg_scorestudent_count
A74.33
B93.52
C881

그룹으로 묶은 결과 자체에 다시 조건을 걸고 싶다면 WHERE가 아니라 HAVING을 사용합니다.

googlesql
SELECT class, AVG(score) AS avg_score
FROM `myproject.mydataset.students`
GROUP BY class
HAVING AVG(score) >= 80;
⚠️

WHERE는 그룹으로 묶기 전 개별 행을 거르고, HAVING은 그룹으로 묶은 후 결과를 거릅니다. 헷갈리기 쉬운 부분이니 꼭 기억해두세요.

STEP 05 · 정렬

ORDER BY / LIMIT — 정렬과 개수 제한

ORDER BY는 결과를 정렬하고, LIMIT은 보여줄 결과 개수를 제한합니다.

googlesql
SELECT name, score
FROM `myproject.mydataset.students`
ORDER BY score DESC
LIMIT 3;
namescore
하은95
서연92
수아88
키워드의미
ASC오름차순 (작은 값 → 큰 값, 기본값이라 생략 가능)
DESC내림차순 (큰 값 → 작은 값)
💰

BigQuery 실전 팁 — LIMIT은 화면에 보여지는 행 수만 줄일 뿐, 실제로 스캔하는 데이터 양(과금 기준)을 줄여주지는 않습니다. 비용을 아끼려면 WHERE로 미리 범위를 좁히는 게 중요해요.

참고 · 기호 정리

따옴표 3총사 — ' " ` 는 뭐가 다른가요?

SQL을 보다 보면 작은따옴표, 큰따옴표, 백틱(`) 세 가지가 비슷하게 생겨서 헷갈립니다. 역할이 완전히 다릅니다.

기호이름용도주로 쓰는 곳
'단어'작은따옴표문자열 값을 감쌀 때모든 SQL 공통
"단어"큰따옴표식별자(테이블/컬럼명)를 감쌀 때표준 SQL, PostgreSQL
`단어`백틱식별자(테이블/컬럼명)를 감쌀 때MySQL, BigQuery(GoogleSQL)

즉 '서울'처럼 작은따옴표는 값을 나타내고, 백틱은 이름(식별자)이 예약어와 겹치거나 특수문자·공백을 포함할 때 "이건 진짜 이름이야"라고 컴퓨터에게 알려주는 역할을 합니다.

BigQuery에서는 특히 더 자주 씁니다

BigQuery는 테이블을 프로젝트.데이터셋.테이블 전체 경로로 참조하는데, 프로젝트 ID에 하이픈(-)이 포함된 경우 백틱 없이는 쿼리가 깨집니다. 업로드해주신 화면의 예시가 정확히 이 케이스입니다.

googlesql
SELECT *
FROM `fastfive-data-mart.cms.branch`
WHERE address LIKE '%강남%';
💡

fastfive-data-mart처럼 프로젝트 ID에 하이픈이 들어가면, 백틱 없이 그냥 쓸 경우 BigQuery는 이걸 "fastfive 빼기 data 빼기 mart" 같은 계산식으로 잘못 해석합니다. 그래서 전체 경로를 백틱으로 통째로 감싸주는 것이 BigQuery의 관례입니다.

종합

전체 조합 예제

지금까지 배운 모든 키워드를 한 문장에 합쳐보겠습니다.

googlesql
SELECT
  s.class,
  COUNT(DISTINCT c.club_name) AS club_count,
  AVG(s.score) AS avg_score
FROM `myproject.mydataset.students` AS s
LEFT JOIN `myproject.mydataset.clubs` AS c
  ON s.id = c.student_id
WHERE s.city != '부산'
GROUP BY s.class
HAVING AVG(s.score) >= 80
ORDER BY avg_score DESC
LIMIT 2;

우리말로 풀면: "부산이 아닌 학생들을, 동아리 정보와 함께 연결해서, 반별로 묶고, 평균 80점 이상인 반만 남긴 뒤, 평균이 높은 순으로 정렬해 상위 2개만 보여줘."

참고 · 데이터 타입

BigQuery에서 자주 쓰는 데이터 타입

테이블의 각 열(컬럼)은 정해진 데이터 타입을 가집니다. 타입을 알아야 어떤 연산(계산, 비교, 정렬)이 가능한지 알 수 있습니다.

타입분류설명예시 값
STRING문자형문자열 텍스트'강남점'
INT64숫자형정수1, 100, -5
FLOAT64숫자형실수 (부동소수점)3.14
NUMERIC숫자형정밀한 소수 (금액 등)1234.56
DATE날짜/시간날짜만2026-06-19
DATETIME날짜/시간날짜+시간 (타임존 없음)2026-06-19 14:30:00
TIMESTAMP날짜/시간날짜+시간 (UTC 타임존 포함)2026-06-19T05:30:00Z
BOOL논리형참 / 거짓TRUE, FALSE
ARRAY기타같은 타입 값 여러 개를 담는 리스트[1, 2, 3]
STRUCT기타여러 필드를 묶은 중첩 구조{name:'강남점'}
ℹ️

ARRAY와 STRUCT는 한 셀 안에 여러 값을 중첩해서 담을 수 있는 BigQuery(GoogleSQL)만의 특징적인 타입입니다. 처음에는 STRING / INT64 / FLOAT64 / DATE / TIMESTAMP / BOOL 정도만 알아도 충분합니다.

⚙️ 타입 변환이 필요한 이유

예를 들어 branch_id 컬럼이 STRING으로 저장되어 있다면, ORDER BY로 정렬했을 때 "10"이 "2"보다 먼저 오는 등 숫자가 아니라 문자 순서로 정렬되는 문제가 생깁니다. 이럴 때는 명시적으로 타입을 변환해줘야 합니다.

googlesql
SELECT
  CAST(branch_id AS INT64) AS branch_id_num,
  SAFE_CAST(open_date AS DATE) AS open_date_parsed
FROM `myproject.mydataset.branch`;
함수용도
CAST(값 AS 타입)타입 변환. 변환할 수 없는 값이 있으면 에러 발생
SAFE_CAST(값 AS 타입)타입 변환. 변환할 수 없는 값은 에러 대신 NULL 반환 (더 안전)
PARSE_DATE('%Y%m%d', 값)'20260619' 같은 특정 포맷의 문자열을 DATE로 변환
💡

실무 데이터는 형식이 깨끗하지 않은 경우가 많아서, 에러로 쿼리 전체가 멈추는 CAST보다 SAFE_CAST를 기본으로 쓰는 것을 추천합니다.

연습

직접 풀어보는 연습문제

students / clubs 테이블을 기준으로 아래 질문에 맞는 SQL을 직접 작성해보세요.

1. 이름과 도시만 조회하기 
SELECT name, city FROM `myproject.mydataset.students`;
2. 점수가 80점 미만인 학생 조회하기 
SELECT * FROM `myproject.mydataset.students` WHERE score < 80;
3. 이름에 "도"가 들어가는 학생 찾기 (LIKE 활용) 
SELECT * FROM `myproject.mydataset.students` WHERE name LIKE '%도%';
4. 점수가 높은 순으로 정렬해서 전체 보여주기 
SELECT * FROM `myproject.mydataset.students` ORDER BY score DESC;
5. 도시별 학생 수 구하기 
SELECT city, COUNT(*) AS student_count
FROM `myproject.mydataset.students`
GROUP BY city;
6. 동아리에 가입하지 않은 학생만 찾기 (LEFT JOIN 응용) 
SELECT s.name
FROM `myproject.mydataset.students` AS s
LEFT JOIN `myproject.mydataset.clubs` AS c
  ON s.id = c.student_id
WHERE c.club_name IS NULL;
7. score가 STRING으로 저장되어 있다고 가정하고, 숫자로 변환해서 정렬하기 (CAST 활용) 
SELECT name, CAST(score AS INT64) AS score_num
FROM `myproject.mydataset.students`
ORDER BY score_num DESC;
