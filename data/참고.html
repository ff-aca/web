● BigQuery (GoogleSQL) 기준

올려주신 지점(branch) 테이블로
SQL 기초 다시 정리하기

실제로 쓰시는 branch 테이블 구조를 그대로 예제로 써서, BigQuery 문법까지 한 번에 익히는 가이드입니다.

SELECT› FROM› WHERE› GROUP BY› HAVING› ORDER BY› LIMIT

필요한 키워드만 골라 쓰면 되고, 쓸 때는 반드시 이 순서를 지켜야 합니다.

BigQuery에서 테이블 이름은 3단계 구조

일반 SQL과 가장 큰 차이는 FROM 뒤에 오는 이름입니다. BigQuery는 테이블 하나를 project.dataset.table 형태로 가리킵니다.

fastfive-data-mart프로젝트 (project)
cms데이터셋 (dataset)
branch테이블 (table)

프로젝트 이름에 하이픈(-)처럼 특수문자가 들어있으면, 셋을 합친 전체 이름을 반드시 백틱(`)으로 감싸야 합니다. 하이픈은 일반 식별자에 쓸 수 없는 특수문자라서, project.dataset.table 전체를 백틱으로 감싸야 BigQuery가 "이건 진짜 이름이야"라고 인식합니다.

STEP 01 · 조회

SELECT / FROM — 데이터 조회하기

SELECT는 "어떤 열을 보고 싶은지", FROM은 "어느 테이블에서 가져올지"를 정합니다.

googlesql 
SELECT name, seats
FROM `my-company-data.cms.branch`;

모든 열을 다 보고 싶다면 "전부"라는 뜻의 *(별표)를 씁니다.

googlesql
SELECT *
FROM `my-company-data.cms.branch`;
💰

BigQuery 실전 팁 — SELECT *는 모든 컬럼을 다 읽기 때문에 과금되는 스캔 데이터 양이 늘어납니다. 실무에서는 꼭 필요한 컬럼만 선택하는 습관을 들이는 게 좋아요.

STEP 02 · 조건

WHERE — 조건으로 걸러내기

조건을 만족하는 행만 골라낼 때 사용합니다.

googlesql
SELECT name, seats
FROM `my-company-data.cms.branch`
WHERE seats >= 250;

자주 쓰는 조건 연산자

연산자의미예시
=같다WHERE city = '서울'
!= / <>다르다WHERE sub_brand != 'partner'
> < >= <=크다 / 작다WHERE seats > 200
AND그리고 (둘 다 만족)WHERE city='서울' AND seats>250
OR또는 (하나만 만족)WHERE city='서울' OR city='경기'

🔍 LIKE와 와일드카드 — 보내주신 쿼리가 바로 이거예요

LIKE는 정확히 일치하지 않아도 비슷한 패턴의 문자열을 찾을 때 씁니다. 보내주신 화면의 WHERE address LIKE '%강남%'이 바로 이 문법입니다.

기호의미예시
%0개 이상의 임의의 문자LIKE '%강남%' → "강남"이 어디든 포함
_정확히 1개의 임의의 문자LIKE '강남_호점' → "강남" 다음 1글자 + "호점"

SELECT *

WHERE address LIKE '%강남%';

STEP 03 · 정렬

ORDER BY / LIMIT — 정렬과 개수 제한

ORDER BY는 결과를 정렬하고, LIMIT은 보여줄 결과 개수를 제한합니다.

googlesql
SELECT name, seats
FROM `my-company-data.cms.branch`
ORDER BY seats DESC
LIMIT 3;
nameseats
강남1호점320
강남3호점300
강남2호점280
키워드의미
ASC오름차순 (작은 값 → 큰 값, 기본값이라 생략 가능)
DESC내림차순 (큰 값 → 작은 값)
💰

BigQuery 실전 팁 — LIMIT은 화면에 보여지는 행 수만 줄일 뿐, 실제로 스캔하는 데이터 양(과금 기준)을 줄여주지는 않습니다. 비용을 아끼려면 WHERE로 미리 범위를 좁히는 게 중요해요.

STEP 04 · 집계

GROUP BY — 그룹으로 묶어서 집계하기

같은 값을 가진 행끼리 묶어서, 집계 함수로 합계나 평균을 계산할 때 사용합니다.

함수의미
COUNT()개수
SUM()합계
AVG()평균
MAX() / MIN()최댓값 / 최솟값
googlesql
SELECT city, COUNT(*) AS branch_count, AVG(seats) AS avg_seats
FROM `my-company-data.cms.branch`
GROUP BY city;
citybranch_countavg_seats
서울6238.3
경기1260
인천1220

그룹으로 묶은 결과 자체에 다시 조건을 걸고 싶다면 WHERE가 아니라 HAVING을 사용합니다.

googlesql
SELECT city, AVG(seats) AS avg_seats
FROM `my-company-data.cms.branch`
GROUP BY city
HAVING AVG(seats) >= 230;
⚠️

WHERE는 그룹으로 묶기 전 개별 행을 거르고, HAVING은 그룹으로 묶은 후 결과를 거릅니다. 헷갈리기 쉬운 부분이니 꼭 기억해두세요.

참고 · 기호 정리

따옴표 3총사 — ' " ` 는 뭐가 다른가요?

SQL을 보다 보면 작은따옴표, 큰따옴표, 백틱(`) 세 가지가 비슷하게 생겨서 헷갈립니다. 역할이 완전히 다릅니다.

기호이름용도주로 쓰는 곳
'단어'작은따옴표문자열 값을 감쌀 때모든 SQL 공통
"단어"큰따옴표식별자(테이블/컬럼명)를 감쌀 때표준 SQL, PostgreSQL
`단어`백틱식별자(테이블/컬럼명)를 감쌀 때BigQuery(GoogleSQL), MySQL

BigQuery에서 백틱이 가장 자주 등장하는 이유는 프로젝트 이름에 하이픈(-)이 들어가는 경우가 많기 때문입니다. 하이픈은 일반 식별자에 쓸 수 없는 특수문자라서, project.dataset.table 전체를 백틱으로 감싸야 BigQuery가 "이건 진짜 이름이야"라고 인식합니다.

googlesql
-- my-company-data 처럼 프로젝트 이름에 하이픈(-)이 있어서 백틱이 꼭 필요한 경우
SELECT *
FROM `my-company-data.cms.branch`
WHERE city = '서울';
💡

컬럼명이 order처럼 SQL 예약어와 겹치거나 공백을 포함할 때도 백틱이 필요합니다. 반대로 city, seats처럼 평범한 이름은 굳이 백틱 없이도 잘 동작해요.

전체 조합 예제

지금까지 배운 모든 키워드를 한 문장에 합쳐보겠습니다.

googlesql
SELECT city, AVG(seats) AS avg_seats, COUNT(*) AS branch_count
FROM `my-company-data.cms.branch`
WHERE sub_brand = 'spaceone'
GROUP BY city
HAVING AVG(seats) >= 230
ORDER BY avg_seats DESC
LIMIT 2;
cityavg_seatsbranch_count
경기2601
서울2565

우리말로 풀면: "sub_brand가 spaceone인 지점만 골라서, 도시별로 묶고, 평균 좌석수가 230 이상인 도시만 남긴 뒤, 평균이 높은 순으로 정렬해 상위 2개만 보여줘."

직접 풀어보는 연습문제

branch 테이블을 기준으로 아래 질문에 맞는 SQL을 직접 작성해보세요.

1. name과 city만 조회하기 
SELECT name, city FROM `my-company-data.cms.branch`;
2. 좌석수(seats)가 250 이상인 지점 조회하기 
SELECT * FROM `my-company-data.cms.branch` WHERE seats >= 250;
3. address에 "강남"이 포함된 지점 찾기 (LIKE) 
SELECT * FROM `my-company-data.cms.branch` WHERE address LIKE '%강남%';
4. 좌석수 많은 순으로 정렬해서 전체 보여주기 
SELECT * FROM `my-company-data.cms.branch` ORDER BY seats DESC;
5. sub_brand별 지점 수와 평균 좌석수 구하기 
SELECT sub_brand, COUNT(*) AS branch_count, AVG(seats) AS avg_seats
FROM `my-company-data.cms.branch`
GROUP BY sub_brand;
다음 단계로는 JOIN(여러 테이블 연결), INSERT/UPDATE/DELETE, 서브쿼리를 배워보세요.
